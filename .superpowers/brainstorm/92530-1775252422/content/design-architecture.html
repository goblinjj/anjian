魔力宝贝制造助手 — 整体架构

专用制造自动化工具，内置配方管理

系统架构
┌─────────────────────────────────────────────────────┐
│                   主界面 (main_gui.py)               │
│  ┌──────────────┐  ┌────────────────────────────┐   │
│  │  配方列表     │  │  制造控制面板               │   │
│  │  - 生命力回复药│  │  当前配方信息               │   │
│  │  - 魔力回复药  │  │  [开始制造] [停止]          │   │
│  │  - ...        │  │  状态日志                   │   │
│  │              │  │  成功: 12  失败: 3           │   │
│  │ [新建] [编辑] │  │                             │   │
│  │ [删除]       │  │                             │   │
│  └──────────────┘  └────────────────────────────┘   │
└─────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────┐
│              配方编辑对话框 (recipe_dialog.py)         │
│  配方名称: [生命力回复药(300)]                         │
│  等待时间: [3.0] 秒                                   │
│  材料列表:                                            │
│  ┌─────────┬──────────┬──────────────────┐           │
│  │ 材料图片  │ 需求数量  │ 操作             │           │
│  │ [截图1]  │   15     │ [重新截图] [删除] │           │
│  │ [截图2]  │    5     │ [重新截图] [删除] │           │
│  │ [截图3]  │   10     │ [重新截图] [删除] │           │
│  └─────────┴──────────┴──────────────────┘           │
│  [添加材料]                    [保存] [取消]           │
└──────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────┐
│                  后端模块                              │
│                                                      │
│  backpack_reader.py    背包识别                        │
│  ├─ 自动定位背包窗口（识别边框/标题"物品"）              │
│  ├─ 切割 5×4 网格 → 20 个格子                         │
│  ├─ 每格提取物品图标区域 + 数量区域                     │
│  └─ 返回: [{icon_image, quantity, grid_pos}, ...]     │
│                                                      │
│  digit_recognizer.py   数字识别                        │
│  ├─ 内置 0-9 数字模板图片                              │
│  ├─ 从格子右下角区域提取数字                            │
│  └─ 模板匹配 → 返回整数 (1-80)                        │
│                                                      │
│  craft_engine.py       制造引擎                        │
│  ├─ 输入: 配方(材料图片+数量) + 背包扫描结果            │
│  ├─ 匹配: 找图标相似且数量≥需求的格子                   │
│  ├─ 执行: 点击格子 → 点击执行 → 等待 → 检测完成        │
│  └─ 循环: 制造完成后重新扫描背包，继续制造              │
│                                                      │
│  recipe_manager.py     配方管理                        │
│  ├─ 配方的增删改查                                     │
│  ├─ 持久化到 recipes/ 目录 (JSON + 材料图片)           │
│  └─ 每个配方一个子目录                                 │
│                                                      │
│  screenshot_util.py    截图工具 (复用现有)              │
└──────────────────────────────────────────────────────┘

制造执行流程

开始制造
  │
  ▼
┌─────────────────┐
│ 1. 定位背包窗口  │ ← 识别"物品"标题栏
└────────┬────────┘
         ▼
┌─────────────────┐
│ 2. 扫描背包      │ ← 切割20格，识别图标+数量
└────────┬────────┘
         ▼
┌─────────────────┐
│ 3. 匹配材料      │ ← 对每种材料找 图标匹配 且 数量≥需求 的格子
└────────┬────────┘
     匹配成功？
    ╱        ╲
  是           否 → 状态栏提示"材料不足"，等待用户处理
   │
   ▼
┌─────────────────┐
│ 4. 依次点击格子   │ ← 按顺序点击匹配到的格子
└────────┬────────┘
         ▼
┌─────────────────┐
│ 5. 点击执行按钮   │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 6. 等待制造完成   │ ← 可配置等待时间 + 检测完成按钮
└────────┬────────┘
         ▼
┌─────────────────┐
│ 7. 点击完成按钮   │
└────────┬────────┘
         ▼
    回到步骤 2 ──── 循环直到用户手动停止
