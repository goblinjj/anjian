# 魔力宝贝制造助手 — 设计文档

## 概述

将现有的通用自动化工具（按键小精灵）改造为魔力宝贝游戏专用的制造自动化工具。核心功能是智能识别背包物品及数量，根据配方需求自动选择材料并执行制造，支持循环制造。

### 核心问题

现有工具仅通过图片识别来选择道具，无法识别道具数量。当背包中某物品数量不足配方需求时，程序会反复点击该格子导致卡死流程。

### 解决方案

开发专用的制造助手，具备：
- 背包物品图标识别 + 数量识别（模板匹配）
- 智能材料选择（图标匹配 + 数量 >= 需求）
- 配方管理（保存/加载配方，一键制造）
- 游戏窗口绑定（支持多开，通过点击选择目标窗口）

### 窗口绑定流程

1. 用户点击 [点击选择窗口]
2. 工具窗口最小化，光标变为十字
3. 用户点击目标游戏窗口的任意位置
4. 程序通过 `win32gui.WindowFromPoint` 获取该窗口句柄
5. 显示绑定信息，后续所有操作基于该句柄定位
6. 通过句柄实时获取窗口坐标，窗口移动后仍然正确

多开场景：所有游戏窗口标题相同，因此使用点击选择而非标题匹配来绑定具体窗口。

### 模块职责

**window_manager.py — 窗口管理**
- 点击选择模式：最小化工具窗口 → 监听鼠标点击 → `WindowFromPoint` 获取窗口句柄
- 通过句柄获取窗口实时坐标：`GetWindowRect(hwnd)`
- 坐标转换：窗口相对坐标 ↔ 屏幕绝对坐标
- 依赖：`win32gui`, `win32api`

**backpack_reader.py — 背包识别**
- 在绑定窗口截图内，用模板匹配定位背包窗口（匹配"物品"标题栏图片）
- 从背包标题栏位置推算网格起始点，按固定像素间距切割 5列×4行 = 20个格子
- 每个格子提取：物品图标区域 + 右下角数量区域
- 返回结构：`[{icon_image, quantity, grid_position}, ...]`
- 依赖：`screenshot_util`, `digit_recognizer`, OpenCV

**digit_recognizer.py — 数字识别**
- 加载 `templates/digits/` 下的 0-9 模板图片
- 从格子右下角区域提取数字：模板逐位匹配
- 数字范围 1-80（1-2位数）
- 没有数字的格子（不可堆叠物品）返回 None
- 依赖：OpenCV, numpy

**craft_engine.py — 制造引擎**
- 输入：配方（材料图片列表 + 各自需求数量）+ 背包扫描结果
- 材料匹配：对每种材料，在背包中找到图标匹配且数量 >= 需求的格子
- 制造循环：扫描 → 匹配 → 点击格子 → 点击执行 → 等待 → 检测完成 → 点击完成 → 整理（按频率）→ 重复
- 材料不足时：日志提示，暂停等待用户处理
- 运行在守护线程中，通过 `should_stop` 标志取消
- 依赖：`backpack_reader`, `window_manager`, `pyautogui`

**recipe_manager.py — 配方管理**
- 配方的增删改查
- 每个配方存储为 `recipes/{配方名}/` 目录：`recipe.json` + 材料截图 PNG
- `recipe.json` 结构：`{name, wait_time, organize_interval, materials: [{image_file, quantity}, ...]}`

**recipe_dialog.py — 配方编辑对话框**
- tkinter 对话框：配方名称、等待时间、整理频率、材料列表
- 材料截图功能：最小化窗口 → 用户框选 → 保存截图
- 新建/编辑模式

**main_gui.py — 主界面（重写）**
- 窗口绑定栏：[点击选择窗口] 按钮 + 绑定状态显示
- 配方列表面板：列表 + 新建/编辑/删除按钮
- 制造控制面板：配方详情 + 开始/停止 + 运行日志 + 统计
- 设置按钮：打开全局设置对话框
- 状态栏：当前状态 + 热键提示

## 技术要点

- **坐标体系**：所有操作基于窗口句柄实时获取坐标，窗口移动后仍然正确
- **物品识别**：OpenCV 模板匹配，格子中的物品图标 vs 配方材料截图
- **数量识别**：0-9 数字模板逐位匹配，数字范围 1-80，没有数字返回 None（不可堆叠物品）
- **目标平台**：Windows，依赖 `win32gui`/`win32api` 进行窗口操作
- **GUI 框架**：tkinter（与现有项目一致）
- **关键依赖**：opencv-python, pyautogui, numpy, Pillow, mss, pywin32, keyboard
